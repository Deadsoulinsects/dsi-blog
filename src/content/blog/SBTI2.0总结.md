# 过程
九九八十一难第一难：**安装**。
纯小白没接触过这些，什么都不懂，先是安装了opencode的桌面端，后来看评论，说桌面端有很多bug，目前阶段不推荐用。于是我就去下终端版，没想到这node.js非常难搞，具体细节忘了，反正是花了不少时间在配置环境上。嗐，由于我是搞完之后才写的分享，所以忘了好多内容。

第二步就是借助AI完成测试网页的内容，没什么好讲的，反复调试就行。以现在来看，可以总结点小技巧：
1. 由于我喜欢先苦后甜，所以我什么都没变，用的原生opencode和免费模型，如果你也想玩，建议优化一下opencode
2. 不要上来就干活，正所谓磨刀不误砍柴工，先给总提示词奠定基调很重要
3. 适当保留测试文件。并且要整合测试文件，不要让文件太乱
4. 不要让AI直接进行改动，先给出设计理念（可以用HTML展示），确定好再做改动

最后就是**部署网站**了，我先是用的[netlify](https://www.netlify.com/)（免费）部署，但是由于没什么经验，导致我最终用完了免费额度也没做好最终版本，因为我是部署之后才进行手机端测试，手机端适配又微调了很多次。最后额度用完之后又调整了几次我才做好了差强人意的最终版。但是没额度我又部署不了，只好用服务器建个真正的网站了，这里再说一下这个经历吧

用的是腾讯云“轻量应用服务器”，这个有陷阱啊：租服务器的时候建议问下腾讯云内置AI，因为AI会给你不显示的购买方案，我这个就是，不仅更便宜而且性能还好点。买完之后在服务器内上传你的网站文件，然后借助AI用nginx部署网站，到这一步其实已经结束了，但是此时访问地址是你的服务器公网IP，不太美观，所以我又买了域名，解析完之后就得到了最终的网址。哦对，别忘了给网站备案哦
