这次决定用 Astro 配合 Markdown 来搭这个站，原因很直接：它足够轻，也足够清楚。页面结构、内容来源、样式边界，基本都能分得比较明白。对于还在起步阶段的个人博客来说，这种清楚感很重要，因为它能减少很多不必要的犹豫。不是每一次建站都需要从复杂方案开始，有时候先把文章稳定放出来，比功能堆叠更有意义。

Markdown 的好处也很明显。写作时不用分心去处理太多界面操作，标题、段落、列表这些最常用的内容都很直接。本地文件本身也比较容易管理，迁移和备份都轻松一些。对我来说，这种方式像是在写一份整理过的笔记，而不是在操作一个必须时刻在线的后台。它保留了写字时需要的专注，也让内容和站点之间的连接更简单。

Astro 在这里扮演的角色很合适。它把内容读取、路由生成和页面输出这几件事处理得比较自然，站点整体不会显得笨重。对于博客来说，读者真正关心的还是打开速度、排版体验和信息是否清楚，而不是背后用了多少层结构。能让内容平稳地呈现出来，其实已经完成了大部分任务。

当然，这种方式也提醒我一件事：建站并不只是把技术跑通。真正决定一个博客是否能继续写下去的，往往是内容整理流程是否顺手。Astro 和 Markdown 只是把入口搭好了，接下来更重要的是维持节奏，持续补上真正想写的东西。等这些文章逐渐积累起来，这个站才会慢慢有自己的重量。
